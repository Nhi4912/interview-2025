## 🎯 Interview Process Overview

```
┌─────────────────────────────────────────────────────────────────┐
│                    AMAZON INTERVIEW PROCESS                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│   1. ONLINE ASSESSMENT (OA)                                     │

│   ⚠️ Bar Raiser can veto any candidate regardless of            │
│   technical performance!                                        │
```

## 📜 Leadership Principles Deep Dive

### Top Priority LPs (Always Asked)

```
┌─────────────────────────────────────────────────────────────────┐
│                    MUST-PREPARE LPs                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│   1. CUSTOMER OBSESSION                                         │
│      "Start with customer and work backwards"                   │
│      Q: "Tell me about putting customer first"                  │
│      Q: "Describe a decision that prioritized customer"         │
│      Story: User research → feature change → customer impact    │
│                                                                   │
│   2. OWNERSHIP                                                  │
│      "Never say 'that's not my job'"                           │
│      Q: "Tell me about going beyond your role"                  │
│      Q: "Describe taking ownership of something"                │
│      Story: Identified problem → took action → long-term result │
│                                                                   │
│   3. DIVE DEEP                                                  │
│      "Leaders operate at all levels, stay connected to details" │
│      Q: "Tell me about debugging a complex issue"               │
│      Q: "How do you ensure you understand the details?"         │
│      Story: Surface problem → investigation → root cause        │
│                                                                   │
│   4. DELIVER RESULTS                                            │
│      "Focus on key inputs and deliver"                          │
│      Q: "Tell me about delivering under pressure"               │
│      Q: "Describe meeting a difficult deadline"                 │
│      Story: Challenge → actions → quantified result             │
│                                                                   │
│   5. EARN TRUST                                                 │
│      "Listen, speak candidly, treat others respectfully"        │
│      Q: "Tell me about building trust with a teammate"          │
│      Q: "Describe a time you admitted a mistake"                │
│      Story: Conflict → honest conversation → improved relation  │
│                                                                   │
└─────────────────────────────────────────────────────────────────┘
```

### Secondary LPs (Frequently Asked)

```
BIAS FOR ACTION
"Speed matters in business"
Story: Quick decision with incomplete info

INSIST ON HIGHEST STANDARDS
"Leaders have relentlessly high standards"
Story: Pushing for quality when others compromised

HAVE BACKBONE; DISAGREE AND COMMIT
"Respectfully challenge decisions when you disagree"
Story: Pushed back on a decision → outcome

LEARN AND BE CURIOUS
"Never done learning"
Story: Learning new technology → applying it

INVENT AND SIMPLIFY
"Expect and require innovation"
Story: Creative solution to a problem
```

### LP Interview Format

```
EVERY round includes LPs:
• 2-3 LP questions per round
• STAR method is REQUIRED
• Expect follow-up questions:
  - "What would you do differently?"
  - "What was the outcome?"
  - "How did others react?"
  - "What did you learn?"

PREPARE:
• 2+ stories per top-5 LP
• 1 story per secondary LP
• Stories should be adaptable
```

## 📝 STAR Example for Amazon

```
QUESTION: "Tell me about a time you went above and beyond"
(Testing: Customer Obsession + Ownership)

SITUATION:
"At my previous company, a major client reported checkout
failures. Support escalated to engineering, but it wasn't
assigned to anyone yet."

TASK:
"Though it wasn't my feature, I knew the client represented
$2M ARR. I volunteered to investigate immediately."

ACTION:
"I analyzed logs and found a race condition in our payment
webhook. I:
1. Wrote a hotfix within 2 hours
2. Coordinated with the payment team for testing
3. Deployed same-day with rollback plan
4. Documented the issue for the team"

RESULT:
"The fix reduced checkout failures from 5% to 0.1%.
The client stayed and renewed for 2 years.
My solution became the standard pattern for webhook handling."

LEARNINGS:
"This taught me to always consider customer impact when
prioritizing work, even if something isn't 'my job'."
```
