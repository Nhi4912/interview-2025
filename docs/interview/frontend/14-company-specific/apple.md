# Apple Frontend Interview Guide

> Apple đề cao attention to detail và passion for products. Interview process dài nhất trong Big Tech.

---

## 🎯 Interview Process Overview

```
┌─────────────────────────────────────────────────────────────────┐
│                    APPLE INTERVIEW PROCESS                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│   1. RECRUITER SCREEN (30 min)                                  │
│      • Resume review                                            │
│      • Basic qualifications                                     │
│      • Role overview                                            │
│                                                                   │
│   2. TECHNICAL PHONE SCREEN (45-60 min)                         │
│      • Technical questions                                      │
│      • May include coding                                       │
│                                                                   │
│   3. ONSITE LOOP (6-8 rounds - LONGEST!)                       │
│      • 3-4 Technical/Coding rounds                              │
│      • 1-2 System Design rounds                                 │
│      • 1 Design Critique round                                  │
│      • 1-2 Behavioral/Team fit rounds                           │
│      • Hiring Manager round                                     │
│                                                                   │
│   4. DEBRIEF                                                    │
│      • All interviewers discuss                                 │
│      • Strong consensus needed                                  │
│                                                                   │
│   5. OFFER                                                      │
│      • Competitive, RSU heavy                                   │
│                                                                   │
│   TIMELINE: 4-8 weeks                                           │
│                                                                   │
│   ⚠️ Apple interviews are EXHAUSTING. Plan for a full day.     │
│   Pace yourself!                                                │
│                                                                   │
└─────────────────────────────────────────────────────────────────┘
```

## 💻 Technical Interviews

### What to Expect

```
MORE TECHNICAL DEPTH THAN OTHER FAANG:
• Multiple coding rounds
• Deep technical questions
• Expect follow-up questions
• Performance optimization

CODING FOCUS:
□ Data structures and algorithms
□ Object-oriented design
□ System design
□ Platform-specific (iOS/macOS sometimes)

FRONTEND SPECIFIC:
□ DOM manipulation
□ CSS animations and transitions
□ Performance optimization
□ Touch interactions
□ Responsive design
```

### Sample Questions

```javascript
// Build a smooth, 60fps carousel
// - Touch/swipe support
// - Hardware acceleration
// - Memory efficient

// Implement a custom scrollbar
// - Matches native feel
// - Accessible
// - Performant

// Create a photo editing canvas
// - Smooth rendering
// - Undo/redo
// - Touch gestures
```

## 📚 Preparation Checklist

```
TECHNICAL (8 weeks):
□ 100+ LeetCode problems
□ Deep OOP understanding
□ Performance optimization
□ Build polished UI components
□ Practice design patterns

DESIGN (4 weeks):
□ Read Apple HIG thoroughly
□ Analyze Apple apps
□ Practice design critique
□ Build smooth animations
□ Study Apple's design philosophy

SYSTEM DESIGN (3 weeks):
□ Media streaming patterns
□ Offline-first architecture
□ Privacy-preserving design
□ Cross-device sync

BEHAVIORAL (2 weeks):
□ Why Apple (genuine answer!)
□ Passion for products
□ Excellence/detail stories
□ Research team you're applying to
□ Prepare thoughtful questions

STAMINA (1 week):
□ Practice for long interview days
□ Mental preparation
□ Physical preparation (sleep, nutrition)
□ Plan breaks and snacks
```

## 💡 Apple-Specific Tips

```
DO:
✓ Show genuine passion for Apple products
✓ Pay attention to small details in your code
✓ Discuss UX impact of decisions
✓ Consider privacy in designs
✓ Write clean, elegant solutions
✓ Be prepared for long interview day

DON'T:
✗ Fake enthusiasm for Apple
✗ Write messy or quick-fix code
✗ Ignore design/UX aspects
✗ Share confidential info about current job
✗ Rush through problems
✗ Forget about edge cases
```
