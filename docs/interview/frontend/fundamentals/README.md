# Frontend Fundamentals / Nền tảng Frontend

## Overview / Tổng quan

**English:** This section covers the foundational knowledge every frontend engineer should master, including algorithms, data structures, web fundamentals, and more. These concepts form the backbone of successful frontend engineering interviews at Big Tech companies.

**Tiếng Việt:** Phần này bao gồm kiến thức nền tảng mà mọi kỹ sư frontend nên thành thạo, bao gồm thuật toán, cấu trúc dữ liệu, nền tảng web và nhiều hơn nữa. Những khái niệm này tạo thành xương sống của các cuộc phỏng vấn kỹ thuật frontend thành công tại các công ty công nghệ hàng đầu.

## Learning Path / Lộ trình học tập

### English Learning Path:
1. **Week 1-2**: Master JavaScript fundamentals and closures
2. **Week 3**: Deep dive into DOM manipulation and event handling
3. **Week 4**: Understand Event Loop and asynchronous programming
4. **Week 5**: Learn browser internals and rendering pipeline
5. **Week 6**: Focus on performance optimization techniques
6. **Week 7**: Practice algorithms and data structures
7. **Week 8**: Integration and interview preparation

### Lộ trình học (Tiếng Việt):
1. **Tuần 1-2**: Thành thạo nền tảng JavaScript và closures
2. **Tuần 3**: Tìm hiểu sâu về thao tác DOM và xử lý sự kiện
3. **Tuần 4**: Hiểu Event Loop và lập trình bất đồng bộ
4. **Tuần 5**: Học nội bộ trình duyệt và pipeline rendering
5. **Tuần 6**: Tập trung vào kỹ thuật tối ưu hiệu suất
6. **Tuần 7**: Luyện tập thuật toán và cấu trúc dữ liệu
7. **Tuần 8**: Tích hợp và chuẩn bị phỏng vấn

## Success Metrics / Chỉ số thành công

### English Metrics:
- **Technical Depth**: Ability to explain concepts from first principles
- **Problem Solving**: Apply fundamentals to solve complex problems
- **Communication**: Articulate technical concepts clearly
- **Code Quality**: Write clean, efficient, and maintainable code
- **Performance Awareness**: Optimize for real-world constraints

### Chỉ số (Tiếng Việt):
- **Độ sâu kỹ thuật**: Khả năng giải thích khái niệm từ nguyên lý cơ bản
- **Giải quyết vấn đề**: Áp dụng nền tảng để giải quyết vấn đề phức tạp
- **Giao tiếp**: Trình bày khái niệm kỹ thuật một cách rõ ràng
- **Chất lượng code**: Viết code sạch, hiệu quả và có thể bảo trì
- **Nhận thức hiệu suất**: Tối ưu cho các ràng buộc thực tế
