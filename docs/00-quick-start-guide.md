## Interview Types

### 1. Coding Interview
**Format:** Solve algorithmic problems
**Duration:** 45-60 minutes
**Preparation:**
- Practice on LeetCode/HackerRank
- Focus on arrays, strings, trees
- Learn common patterns
- Practice explaining solutions

**Tips:**
- Think out loud
- Ask clarifying questions
- Start with brute force
- Optimize iteratively
- Test your code

### 2. Frontend Coding
**Format:** Build UI components or features
**Duration:** 60-90 minutes
**Preparation:**
- Practice building common components
- Know vanilla JS and React
- Understand CSS layouts
- Handle edge cases

**Common Tasks:**
- Autocomplete
- Infinite scroll
- Modal/Dialog
- Form validation
- Data table

### 3. System Design
**Format:** Design scalable frontend systems
**Duration:** 45-60 minutes
**Preparation:**
- Study common architectures
- Learn scalability patterns
- Understand trade-offs
- Practice explaining designs

**Common Questions:**
- Design a news feed
- Design a chat application
- Design an autocomplete system
- Design a photo sharing app

### 4. Behavioral Interview
**Format:** Discuss experiences and scenarios
**Duration:** 30-45 minutes
**Preparation:**
- Prepare STAR stories
- Review past projects
- Practice common questions
- Be authentic

**Common Questions:**
- Tell me about yourself
- Describe a challenging project
- How do you handle conflicts?
- Why this company?

## Common Mistakes to Avoid

### During Preparation
❌ Jumping to hard problems too early
❌ Not practicing consistently
❌ Ignoring fundamentals
❌ Only watching tutorials
❌ Not building projects
❌ Skipping mock interviews

### During Interview
❌ Not asking clarifying questions
❌ Jumping to code immediately
❌ Not communicating thought process
❌ Giving up too quickly
❌ Not testing code
❌ Ignoring edge cases

## Interview Day Tips

### Before Interview
- Review key concepts
- Get good sleep
- Prepare questions to ask
- Test equipment (for remote)
- Arrive/login early

### During Interview
- Stay calm and confident
- Think out loud
- Ask questions
- Start simple, then optimize
- Test your solution
- Be honest about what you don't know

### After Interview
- Send thank you email
- Reflect on performance
- Note areas to improve
- Follow up appropriately
