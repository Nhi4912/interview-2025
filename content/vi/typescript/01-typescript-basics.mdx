# Kiến thức cơ bản TypeScript

## Giới thiệu

TypeScript là một superset của JavaScript được phát triển bởi Microsoft. Nó thêm static typing vào JavaScript, giúp phát hiện lỗi sớm hơn và cải thiện trải nghiệm phát triển.

## Tại sao sử dụng TypeScript?

1. **Type Safety**: Phát hiện lỗi tại compile time thay vì runtime
2. **Better IDE Support**: Autocomplete, refactoring, và navigation tốt hơn
3. **Self-documenting Code**: Types làm code dễ hiểu hơn
4. **Scalability**: Dễ dàng maintain code base lớn
5. **Modern JavaScript Features**: Hỗ trợ ES6+ features


### Arrays

<CodeExample title="Array Types" language="typescript">
```typescript

let scores: Array<number> = [90, 85, 95];

// Array với nhiều types (Union)

// Readonly array
````

## Interfaces

Interfaces định nghĩa cấu trúc của objects.

<CodeExample title="Interface Examples" language="typescript">
```typescript
interface User {
  id: number;
  name: string;
  email: string;
  age?: number; // Optional
  readonly createdAt: Date; // Readonly
}

const user: User = {
id: 1,
name: "Nguyễn Văn A",
email: "a@example.com",
createdAt: new Date()
};

// Interface với methods
interface Calculator {
add(a: number, b: number): number;
subtract(a: number, b: number): number;
}

const calc: Calculator = {
add: (a, b) => a + b,
subtract: (a, b) => a - b
};

````
</CodeExample>

### Interface Extension

<CodeExample title="Extending Interfaces" language="typescript">
```typescript
interface Person {
  name: string;
  age: number;
}

interface Employee extends Person {
  employeeId: string;
  department: string;
}

const employee: Employee = {
  name: "Nguyễn Văn A",
  age: 25,
  employeeId: "EMP001",
  department: "Engineering"
};

// Multiple inheritance
interface Timestamped {
  createdAt: Date;
  updatedAt: Date;
}

interface Product extends Person, Timestamped {
  price: number;
}
````

</CodeExample>

## Type Aliases

Type aliases tạo tên mới cho types.

<CodeExample title="Type Aliases" language="typescript">
```typescript
// Basic type alias
type ID = string | number;

type User = {
id: ID;
name: string;
email: string;
};

// Union types
type Status = "pending" | "approved" | "rejected";

function updateStatus(status: Status) {
console.log(`Status: ${status}`);
}

updateStatus("approved"); // ✅
// updateStatus("invalid"); // ❌ Error

// Intersection types
type Draggable = {
drag(): void;
};

type Resizable = {
resize(): void;
};

type UIWidget = Draggable & Resizable;

const widget: UIWidget = {
drag() {
console.log("Dragging...");
},
resize() {
console.log("Resizing...");
}
};

````
</CodeExample>

## Functions

<CodeExample title="Function Types" language="typescript">
```typescript
// Function với type annotations
function add(a: number, b: number): number {
  return a + b;
}

// Arrow function
const multiply = (a: number, b: number): number => a * b;

// Optional parameters
function greet(name: string, greeting?: string): string {
  return `${greeting || "Hello"}, ${name}!`;
}

// Default parameters
function createUser(name: string, role: string = "user") {
  return { name, role };
}

// Rest parameters
function sum(...numbers: number[]): number {
  return numbers.reduce((total, n) => total + n, 0);
}

// Function type
type MathOperation = (a: number, b: number) => number;

const divide: MathOperation = (a, b) => a / b;
````

</CodeExample>

## Union và Intersection Types

<CodeExample title="Union and Intersection" language="typescript">
```typescript
// Union type: có thể là một trong nhiều types
type StringOrNumber = string | number;

function printId(id: StringOrNumber) {
if (typeof id === "string") {
console.log(`ID: ${id.toUpperCase()}`);
} else {
console.log(`ID: ${id}`);
}
}

// Intersection type: kết hợp nhiều types
type Person = {
name: string;
age: number;
};

type Employee = {
employeeId: string;
department: string;
};

type EmployeePerson = Person & Employee;

const emp: EmployeePerson = {
name: "An",
age: 25,
employeeId: "E001",
department: "IT"
};

````
</CodeExample>

## Type Assertions

<CodeExample title="Type Assertions" language="typescript">
```typescript
// Cách 1: as syntax
let someValue: unknown = "this is a string";
let strLength: number = (someValue as string).length;

// Cách 2: angle-bracket syntax (không dùng trong JSX)
let strLength2: number = (<string>someValue).length;

// Practical example
const input = document.getElementById("myInput") as HTMLInputElement;
input.value = "Hello";

// Non-null assertion
function processValue(value: string | null) {
  // Chúng ta chắc chắn value không null
  console.log(value!.toUpperCase());
}
````

</CodeExample>

## Type Guards

<CodeExample title="Type Guards" language="typescript">
```typescript
// typeof type guard
function printValue(value: string | number) {
  if (typeof value === "string") {
    console.log(value.toUpperCase());
  } else {
    console.log(value.toFixed(2));
  }
}

// instanceof type guard
class Dog {
bark() {
console.log("Woof!");
}
}

class Cat {
meow() {
console.log("Meow!");
}
}

function makeSound(animal: Dog | Cat) {
if (animal instanceof Dog) {
animal.bark();
} else {
animal.meow();
}
}

// Custom type guard
interface Fish {
swim(): void;
}

interface Bird {
fly(): void;
}

function isFish(pet: Fish | Bird): pet is Fish {
return (pet as Fish).swim !== undefined;
}

function move(pet: Fish | Bird) {
if (isFish(pet)) {
pet.swim();
} else {
pet.fly();
}
}

````
</CodeExample>

## Câu hỏi phỏng vấn thường gặp

### Câu 1: TypeScript là gì và tại sao nên sử dụng?

**Trả lời:**
TypeScript là một superset của JavaScript thêm static typing. Lợi ích:
- **Type Safety**: Phát hiện lỗi sớm hơn
- **Better Tooling**: IDE support tốt hơn
- **Code Documentation**: Types tự document code
- **Refactoring**: Dễ dàng refactor code
- **Scalability**: Tốt cho dự án lớn

### Câu 2: Sự khác biệt giữa Interface và Type Alias?

**Trả lời:**

| Interface | Type Alias |
|-----------|------------|
| Có thể extend | Có thể dùng union/intersection |
| Có thể merge declarations | Không thể merge |
| Chỉ cho objects | Cho mọi type |
| Tốt cho OOP | Linh hoạt hơn |

```typescript
// Interface
interface User {
  name: string;
}

interface User {
  age: number; // Merge với declaration trên
}

// Type
type ID = string | number; // Union type
type Point = { x: number; y: number };
````

### Câu 3: any vs unknown khác nhau như thế nào?

**Trả lời:**

- **any**: Tắt type checking hoàn toàn, không an toàn
- **unknown**: Type-safe, phải check type trước khi sử dụng

```typescript
let anyValue: any = "hello";
anyValue.toUpperCase(); // ✅ Compile OK (có thể error runtime)

let unknownValue: unknown = "hello";
// unknownValue.toUpperCase(); // ❌ Error
if (typeof unknownValue === "string") {
  unknownValue.toUpperCase(); // ✅ OK sau khi check
}
```

### Câu 4: Type assertion là gì?

**Trả lời:**
Type assertion cho phép bạn override type mà TypeScript infer:

```typescript
let value: unknown = "hello";
let length = (value as string).length;

// Hoặc
let length2 = (<string>value).length;
```

Chỉ nên dùng khi bạn chắc chắn về type.

### Câu 5: Readonly vs const khác nhau như thế nào?

**Trả lời:**

- **const**: Cho variables, không thể reassign
- **readonly**: Cho properties, không thể modify sau khi khởi tạo

```typescript
const x = 10;
// x = 20; // ❌ Error

interface Config {
  readonly apiUrl: string;
}

const config: Config = { apiUrl: "https://api.com" };
// config.apiUrl = "new"; // ❌ Error
```

## Best Practices

1. **Tránh any**: Sử dụng unknown hoặc specific types
2. **Sử dụng strict mode**: Enable trong tsconfig.json
3. **Prefer interfaces cho objects**: Dễ extend và merge
4. **Sử dụng type guards**: Thay vì type assertions
5. **Readonly khi có thể**: Immutability tốt hơn
6. **Union types cho enums đơn giản**: Nhẹ hơn enums
7. **Type imports**: Sử dụng `import type` khi chỉ import types
